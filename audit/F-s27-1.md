# F-s27-1: Overview's Pending-JIT panel had no inline Approve/Deny

- Severity: high
- Status: Verified-Fixed
- Area: parity

## Issue

SESSION.md §1.1-C requires the Overview screen's pending-JIT-approvals panel to
offer inline Approve/Deny, matching the design mockup. The rebuilt
`OverviewScreen.tsx` listed pending requests but had no actions column —
identical to `main`, so not a reskin regression, but in-scope work that was
missed during the T2 rebuild.

Found by the T5 design-fidelity + parity reviewer.

## Resolution — Verified-Fixed

`src/features/ir/JitRequestList.tsx` now exports `JitDecisionDialog` and the
`DecisionKind` type so the Overview screen can reuse the exact same
mutation/dialog wiring instead of duplicating it. `OverviewScreen.tsx`'s JIT
columns gained an Actions column with Approve/Deny, gated identically to the
JIT requests screen: `request:approve` permission required, and self-approval
is impossible (`isSelf` check against the signed-in subject) — reflecting
server truth, never offering an action the server would reject.

Regression coverage: `OverviewScreen.test.tsx` — offers Approve/Deny for an
approver, hides both for the requester's own request, hides both without
`request:approve`, and a full approve-and-close-dialog flow against a mocked
`POST /v1/jit-requests/:id/approve`.
