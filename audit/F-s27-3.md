# F-s27-3: mobile nav-toggle glyph doesn't reflect open/closed state

- Severity: low
- Status: Verified-Fixed
- Area: a11y

## Issue

`AppShell.tsx`'s mobile nav-toggle button had correct `aria-label`/
`aria-expanded` wiring for assistive tech, but the visible glyph was hardcoded
to `✕` regardless of `navOpen` — sighted keyboard/mouse users at ≤820px saw a
permanent "close" icon even before opening the nav, misleading about the
actual state (a regression introduced by this reskin; the prior shell showed a
static ☰ with the same limitation, but the new one visually asserts the wrong
state outright).

Found by the T5 a11y auditor.

## Resolution — Verified-Fixed

`{navOpen ? '✕' : '☰'}` — the glyph now tracks `navOpen`, wrapped in an
`aria-hidden="true"` span so the accessible name still comes only from the
`aria-label`.
